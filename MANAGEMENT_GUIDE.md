## 1. 현재 인프라 구성

## 2. 업체(클라이언트)에게 인수인계할 것

### 2-1. 관리자 페이지 사용법 안내

업체 담당자에게 전달할 핵심 내용:

```
관리자 페이지 주소: https://도메인/admin/login
관리자 비밀번호: (설정한 비밀번호)
```

**관리자가 할 수 있는 것:**
- 시공사례(포트폴리오) 등록/수정/삭제 → `/admin/portfolio`
- 인조대리석 샘플 등록/수정/삭제 → `/admin/samples`
- 고객 문의 확인/관리 → `/admin/inquiries`
- 대시보드에서 통계 확인 → `/admin`

**업체 담당자용 간단 매뉴얼 (별도 작성 권장):**
1. 시공사례 등록 방법: 제목, 카테고리 선택, 설명 작성, 사진 여러 장 업로드, 대표 이미지 선택
2. 샘플 등록 방법: 샘플명, 브랜드, 색상, 패턴 분류, 사진 업로드
3. 문의 확인: 새 문의가 들어오면 대시보드에서 확인 가능

### 2-2. 인수인계 체크리스트

- [ ] 관리자 비밀번호 전달 (구두 또는 안전한 방법으로)
- [ ] 관리자 페이지 사용법 시연 (화면 공유 또는 직접 방문)
- [ ] 시공사례 초기 콘텐츠 최소 5~10개 함께 등록
- [ ] 샘플 정보 초기 등록
- [ ] 유지보수 범위 및 비용 합의 (아래 섹션 참고)
- [ ] 연락 채널 확정 (카카오톡, 전화 등)

## 3. 배포 후 정기 관리 항목

### 3-1. 매주 확인 (5분)

| 항목 | 방법 | 왜 필요한가 |
|------|------|------------|
| 사이트 정상 작동 확인 | 브라우저에서 직접 접속 | 서버 오류, 인증서 만료 등 조기 발견 |
| Vercel 대시보드 확인 | Deployments 탭에서 빌드 상태 확인 | 배포 실패 여부 확인 |
| 문의 접수 확인 | `/admin/inquiries` 확인 | 업체가 문의를 놓치지 않도록 |

### 3-2. 매월 확인 (30분)

| 항목 | 방법 | 왜 필요한가 |
|------|------|------------|
| Google Search Console | 검색 실적, 색인 상태 확인 | 검색 노출 추이 모니터링 |
| 네이버 서치어드바이저 | 검색 현황, 크롤링 오류 확인 | 네이버 검색 노출 확인 |
| Vercel Analytics | 방문자 수, 인기 페이지 확인 | 트래픽 추이 파악 |
| Vercel 사용량 | Settings > Usage 확인 | 무료 한도 초과 여부 확인 |
| Turso 사용량 | Turso 대시보드에서 DB 사용량 확인 | Free tier 한도 관리 |

### 3-3. 분기별 확인 (1~2시간)

| 항목 | 방법 | 왜 필요한가 |
|------|------|------------|
| 의존성 보안 업데이트 | `npm audit` 실행, 심각한 취약점 패치 | 보안 유지 |
| Next.js 마이너 업데이트 | `npm outdated`로 확인 후 업데이트 | 버그 수정, 성능 개선 |
| SSL 인증서 확인 | Vercel이 자동 관리하지만 도메인 설정 확인 | HTTPS 유지 |
| 백업 | Turso DB 스냅샷, 이미지 백업 | 데이터 보호 |

### 3-4. 연간 확인

| 항목 | 방법 | 비고 |
|------|------|------|
| 도메인 갱신 | 도메인 등록업체에서 갱신 | 만료되면 사이트 접속 불가 |
| API 키 갱신/확인 | 네이버 지도 API 등 | 키 만료 여부 확인 |
| 서비스 요금제 확인 | Vercel, Turso 등 | 요금 변경 여부 확인 |

## 4. 장애 대응

### 4-1. 사이트 접속이 안 될 때

```
1. Vercel 대시보드 확인 → Deployments에서 최근 배포 상태 확인
2. 도메인 설정 확인 → Settings > Domains에서 DNS 상태 확인
3. Vercel Status Page 확인 → https://www.vercel-status.com/
4. 문제 해결이 안 되면 → Vercel 대시보드에서 이전 배포로 롤백 (Deployments > ... > Promote to Production)
```

### 4-2. 관리자 로그인이 안 될 때

```
1. 비밀번호 확인 → 올바른 비밀번호인지 확인
2. 쿠키 삭제 → 브라우저 쿠키/캐시 삭제 후 재시도
3. 비밀번호 재설정 필요 시:
   - 새 bcrypt 해시 생성: node -e "require('bcryptjs').hash('새비밀번호', 12).then(h => console.log(h))"
   - Vercel 대시보드 > Settings > Environment Variables에서 ADMIN_PASSWORD_HASH 업데이트
   - Redeploy (Deployments > ... > Redeploy)
```

### 4-3. 이미지가 안 보일 때

```
1. Vercel Blob 저장소 확인 → Vercel 대시보드 > Storage > Blob
2. BLOB_READ_WRITE_TOKEN 환경변수 확인
3. 이미지 URL이 유효한지 브라우저에서 직접 접속해 확인
```

### 4-4. DB 관련 오류

```
1. Turso 대시보드에서 DB 상태 확인
2. 환경변수 확인: TURSO_DATABASE_URL, TURSO_AUTH_TOKEN
3. Free tier 한도 초과 여부 확인
```

## 5. 코드 수정/업데이트 방법

### 5-1. 간단한 텍스트/정보 수정

업체 정보(전화번호, 주소 등) 수정이 필요할 때:

```bash
# 1. 로컬에서 코드 수정
# src/lib/constants.ts 파일에서 업체 정보 수정

# 2. 커밋 & 푸시
git add .
git commit -m "업체 정보 수정"
git push

# 3. Vercel이 자동으로 재배포 (2~3분 소요)
```

### 5-2. 기능 추가/수정

```bash
# 1. 새 브랜치 생성
git checkout -b feature/기능명

# 2. 코드 수정

# 3. 로컬 테스트
npm run dev

# 4. 빌드 테스트
npm run build

# 5. 커밋 & 푸시
git add .
git commit -m "feat: 기능 설명"
git push -u origin feature/기능명

# 6. Vercel에서 Preview 배포가 자동 생성됨 → 확인 후 main에 머지
```

### 5-3. 긴급 롤백

문제가 생겼을 때 이전 버전으로 되돌리기:

```
Vercel 대시보드 > Deployments > 정상 작동하던 배포 선택 > ... > Promote to Production
```

## 6. 유지보수 계약 가이드

### 6-1. 유지보수 범위 설정 (예시)

**기본 유지보수 (무료 또는 월 고정 비용):**
- 서버 모니터링 및 장애 대응
- 보안 업데이트 적용
- 간단한 텍스트/이미지 수정 (월 2회 이내)
- 검색엔진 노출 현황 리포트 (월 1회)

**추가 개발 (별도 견적):**
- 새로운 페이지 추가
- 기능 추가 (예: 카카오톡 상담, 이메일 알림 등)
- 디자인 변경
- 성능 최적화

### 6-2. 유지보수 비용 책정 참고

소규모 업체 홍보 사이트 기준 일반적인 시장 가격대:

| 항목 | 가격대 (월) | 포함 내용 |
|------|-------------|-----------|
| 기본 유지보수 | 5~10만원 | 모니터링, 장애 대응, 간단한 수정 |
| 중급 유지보수 | 10~20만원 | 기본 + 콘텐츠 관리 대행, SEO 리포트 |
| 추가 개발 | 시간당 3~5만원 | 기능 추가, 디자인 변경 등 |

> 첫 프로젝트이므로 초기 1~3개월은 무상 유지보수를 제공하고,
> 이후 월 유지보수 계약으로 전환하는 것이 일반적입니다.

### 6-3. 유지보수 계약서에 포함할 내용

1. **유지보수 범위**: 어디까지가 기본이고 어디부터 추가 비용인지 명확히
2. **대응 시간**: 장애 발생 시 몇 시간 이내 대응 (예: 업무일 24시간 이내)
3. **계약 기간**: 월 단위 또는 연 단위
4. **해지 조건**: 1개월 전 통보 등
5. **인프라 비용**: Vercel Pro 전환 시 비용 부담 주체
6. **소스코드 소유권**: 납품 시 소스코드 소유권 이전 여부

## 7. SEO 운영 팁

### 7-1. 검색 노출을 높이려면

1. **시공사례를 꾸준히 올리세요**
   - 새 시공 완료 시마다 사진과 함께 등록
   - 검색엔진은 콘텐츠가 주기적으로 업데이트되는 사이트를 선호합니다
   - 제목에 키워드 포함 (예: "강남 카페 인조대리석 카운터 시공")

2. **Google Search Console 활용**
   - "실적" 탭에서 어떤 키워드로 유입되는지 확인
   - "색인 생성" > "페이지"에서 색인 상태 확인
   - 새 콘텐츠 등록 후 URL 검사 > "색인 생성 요청"으로 빠른 반영

3. **네이버 서치어드바이저 활용**
   - "요청" > "웹 페이지 수집"에서 신규 페이지 수집 요청
   - "현황" > "콘텐츠 노출 현황"에서 검색 키워드 확인

4. **네이버 플레이스 등록** (강력 권장)
   - [네이버 플레이스](https://new-m.place.naver.com/)에 업체 등록
   - "인조대리석 시공" 등으로 지역 검색 시 노출
   - 웹사이트 URL 연결

### 7-2. 초기 색인에 시간이 걸립니다

- 구글: 새 사이트는 색인까지 보통 1~4주 소요
- 네이버: 서치어드바이저에서 수집 요청하면 3~7일 내 반영
- 사이트맵 제출 후 바로 검색되지 않는 것은 정상입니다

## 8. 백업 전략

### 8-1. 소스코드

- GitHub에 자동 백업됨 (git push 시)
- 로컬에도 코드 사본 유지 권장

### 8-2. 데이터베이스

```bash
# Turso CLI로 DB 백업
turso db shell clasic-stone .dump > backup_$(date +%Y%m%d).sql
```

- 월 1회 수동 백업 권장
- 중요 데이터 변경 전후로 백업

### 8-3. 이미지

- Vercel Blob에 저장됨
- 원본 이미지는 업체에서도 별도 보관하도록 안내

## 빠른 참조: 자주 쓰는 명령어

```bash
# 로컬 개발
npm run dev                    # 개발 서버 실행
npm run build                  # 프로덕션 빌드 테스트

# 배포 (main 브랜치에 push하면 자동 배포)
git add .
git commit -m "변경 내용"
git push

# DB 관련
npx drizzle-kit push          # 스키마 변경 적용
npx drizzle-kit studio        # DB GUI 열기

# 의존성 관리

npm outdated                  # 업데이트 가능한 패키지 확인
npm update                    # 패키지 업데이트
```
